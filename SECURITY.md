# Security Policy

## Supported Versions

This project is currently in an early stage of development (`0.1.0`).
Security updates are provided only for the latest version on the default
branch. Older releases and unpublished development snapshots are not
supported with security updates.
## Reporting a Vulnerability

Please report suspected vulnerabilities privately to **security@example.com**.
Do **not** open a public GitHub issue for security reports.

When reporting a vulnerability, please include:
- A description of the issue and its potential impact
- Steps to reproduce the problem
- The affected version(s) and environment, if known
- Any suggested mitigations or fixes, if available

We will acknowledge receipt of your report within **3 business days** and
provide a status update within **7 business days** after acknowledgement.
If the report is accepted, we will work on a fix and coordinate disclosure
with you where appropriate. If the report is declined, we will let you know
why.
